## 📌 О проекте

DetectiveBot — это Telegram-бот, созданный с использованием фреймворка **aiogram 3.x**, который анализирует отправленные контакты и сохраняет их в локальной базе данных с помощью **aiosqlite**(асинхронная версия Sqlite3). С помощью библиотеки **phonenumbers** он определяет страну и сотового оператора контакта, а затем возвращает пользователю аккуратную карточку с информацией.

База данных создается автоматически при запуске — ручная настройка базы данных не требуется.

> Весь текст для пользователей (сообщения, подсказки, ошибки) изначально написан на русском языке в файлах `main.py` и `utils.py`. Если вы хотите сделать английскую версию, вам потребуется перевести строки, передаваемые в `message.answer(...)` / `message.reply(...)`, и переменные в `utils.py`.

## Возможности

| Команда | Альтернатива | Описание |
|---|---|---|
| `/start` | `start` | Приветствует вернувшегося пользователя или просит нового зарегистрироваться, отправив свой контакт. |
| `/help` | `help` | Показывает инструкции по использованию и информацию о перезапуске бота. |
| `/checkuser` | `checkuser` | Просит отправить контакт для поиска информации (основная функция бота). |
| `/share` | `share` | Отправляет QR-код и ссылку, чтобы вы могли пригласить друзей в бота. |
| `/admin` | `admin` | Только для админов. Позволяет владельцу бота (проверяется по `ADMIN_ID`) сделать рекламную рассылку всем зарегистрированным пользователям. |

## Как это работает

### Регистрация и поиск контактов

Когда пользователь отправляет контакт, бот выполняет следующую логику:

1. **Зарегистрирован ли отправитель?** (`user_check` в `data.py`)
2. **Является ли отправленный контакт собственным контактом отправителя?**
   - ✅ **Да, и отправитель зарегистрирован** → бот возвращает собственную сохраненную информацию отправителя.
   - ✅ **Да, и отправитель *не* зарегистрирован** → контакт анализируется и сохраняется; это и *есть* процесс регистрации.
   - ❌ **Нет, это чужой контакт, и отправитель не зарегистрирован** → бот вежливо просит сначала пройти регистрацию, отправив свой собственный контакт.
   - ❌ **Нет, это чужой контакт, и отправитель *зарегистрирован*** → бот проверяет, есть ли этот контакт в базе данных:
     - **Уже в базе** → возвращается сохраненная запись (`get_user` в `data.py`).
     - **Еще нет в базе** → номер анализируется с помощью `phonenumbers` (`search_user_info`), новый контакт сохраняется (`add_user`), и возвращается проанализированная информация.

### Рекламная рассылка (`/admin`)

1. Бот запрашивает у админа текст рекламы.
2. Бот запрашивает ссылку для прикрепления.
3. Сообщение отправляется каждому пользователю в базе данных с инлайн-кнопкой **"Перейти"**. Пользователи, заблокировавшие бота пропускаются.

## Запуск проекта

### 1. Клонирование репозитория

```bash
git clone https://github.com/OAzizjon/DetectiveBot.git
cd DetectiveBot
pip install -r requirements.txt

```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта:

```env
BOT_TOKEN="345676543sdr3"   # Получите у @BotFather
ADMIN_ID="1234567"          # Получите у @GetMyIDo_Bot — код автоматически преобразует строку в int

```

Таблица `users` создается автоматически при первом запуске — вручную настраивать базу данных не нужно.

## Логирование

Все действия и ошибки записываются в файл `app.log` (на уровне DEBUG) и дублируются в консоль (на уровне INFO), что позволяет легко отследить действия любого пользователя и диагностировать сбои.
